# Dowple 대시보드 문서 분류 로직 가이드

본 문서는 Dowple 대시보드 애플리케이션(`dowple_dms.html`)에 적용된 문서 분류(Classification) 로직의 세부 동작 원리를 설명합니다. 현재 적용된 시스템은 순수 AI 모델이 아닌, **'하이브리드 분류(규칙 기반 + 유사도 스코어링 시뮬레이션)'** 방식으로 구현되어 있습니다.

---

## 1. 분류 시스템 개요

문서 업로드 시 파이프라인의 마지막 단계에서 작동하는 분류 엔진은 다음 4가지 단계를 거쳐 최종적으로 3개의 추천 폴더와 신뢰도(%)를 산출합니다.

1. **용어 정규화 (Dictionary Mapping)**
2. **규칙 기반 키워드 매칭 (Rule-based Scoring)**
3. **유사도 및 신뢰도 분배 (Confidence Distribution)**
4. **가비지 문서 예외 처리 (Exception Handling)**

---

## 2. 1단계: 용어 정규화 (Dictionary Mapping)

현업에서 사용되는 다양한 은어, 줄임말, 비표준 용어를 시스템이 이해할 수 있는 **표준 분류어**로 치환하는 전처리 과정입니다. 이는 키워드 매칭의 정확도를 높이기 위한 필수 단계입니다.

* **동작 원리:** 문서 내에 설정된 `DICT` 객체의 Key(은어)가 발견되면 Value(표준어)로 치환합니다.
* **사전 예시:**
  * `찐최종` ➡️ `최종본`
  * `결산안` ➡️ `재무제표`
  * `영수증` ➡️ `회계`
  * `인력관리` ➡️ `인사/총무`

---

## 3. 2단계: 규칙 기반 키워드 매칭 (Rule-based Scoring)

가장 핵심이 되는 점수 산정 단계입니다. 정규화가 완료된 문서 텍스트(제목, AI 요약, 추출된 텍스트 등)를 스캔하여, 사전에 정의된 특정 키워드가 등장할 때마다 해당 폴더에 가중치 점수를 부여합니다.

* **기본 가중치:** 키워드 1개 매칭 시 **+18점**
* **폴더별 매핑 키워드 (`RMAP`):**
  * **재무/회계:** 재무, 회계, 부가세, 재무제표, 세금, 영수증
  * **인사/총무:** 인사, 총무, 채용, 급여, 인력
  * **기술개발:** 개발, 기술, 연구
  * **마케팅/영업:** 마케팅, 영업, 광고, 제안

---

## 4. 3단계: 유사도 및 신뢰도 분배 (Confidence Distribution)

키워드 매칭으로 계산된 점수를 바탕으로 상위 3개의 폴더를 선정하고, AI가 예측한 것과 같은 퍼센티지(%) 형태의 신뢰도 점수를 부여합니다. 이 과정에서 약간의 노이즈(랜덤 변수)가 추가되어 실제 AI 모델처럼 값이 유동적으로 보이도록 시뮬레이션 합니다.

1. **노이즈 추가:** 모든 폴더 점수에 0~4점의 무작위 점수를 더합니다.
2. **정렬 및 추출:** 점수가 가장 높은 상위 3개 폴더를 선정합니다.
3. **신뢰도 곡선 적용:**
   * **1순위 (Top 1):** 기본 점수에 0~7점을 더하여 최고 신뢰도 부여 (최대 95% 제한)
   * **2순위 (Top 2):** 1순위 점수에서 약 20점을 차감하여 격차 생성 (최소 30% 보장)
   * **3순위 (Top 3):** 1순위 점수에서 약 38점을 차감하여 격차 생성 (최소 12% 보장)

---

## 5. 4단계: 가비지 문서 예외 처리 (Exception Handling)

시스템의 정확도를 보완하기 위해 애매한 문서를 걸러내는 안전장치입니다. 스코어링 결과 점수가 낮으면서 동시에 파일명에 임시 파일의 특징이 있는 경우 강제로 '미분류' 처리합니다.

* **발동 조건 (두 가지 모두 충족 시):**
  1. 1순위 추천 폴더의 신뢰도 점수가 **40점 미만**일 때
  2. 파일명에 노이즈 키워드가 포함될 때 (`최종`, `진짜최종`, `수정본`, `v2`, `tmp`)
* **처리 결과:** 1순위 폴더를 **'미분류'**로 변경하고, 점수를 45%로 고정.

---

## 6. 핵심 코드 리뷰 (JavaScript)

위 로직을 관장하는 `dowple_dms.html` 내부의 `classify` 함수입니다.

```javascript
// 하이브리드 분류 엔진 (Rule-based + 시뮬레이션)
function classify(text, fname) {
  var t = text.toLowerCase();
  var scores = [0, 0, 0, 0, 0]; // 각 폴더의 초기 점수

  // 1차: Rule-based 키워드 매칭 (키워드 발견 시 +18점)
  Object.keys(RMAP).forEach(k => {
    if (t.includes(k)) scores[RMAP[k]] += 18;
  });

  // 노이즈 추가 (0~4점 무작위 증가)
  scores = scores.map(s => s + Math.floor(Math.random() * 5));

  // 점수 기준 내림차순 정렬 후 상위 3개 추출
  var sorted = scores
    .map((s, i) => ({ i, s }))
    .sort((a, b) => b.s - a.s)
    .slice(0, 3);

  // 1위 점수 기반으로 2, 3위의 격차 계산
  var base = sorted[0].s || 30; // 일치하는 키워드가 아예 없어도 최소 30점 보장
  var recs = [
    { folder: FOLDERS[sorted[0].i], score: Math.min(95, base + Math.floor(Math.random() * 8)) },
    { folder: FOLDERS[sorted[1].i], score: Math.max(30, base - 20 + Math.floor(Math.random() * 6)) },
    { folder: FOLDERS[sorted[2].i], score: Math.max(12, base - 38 + Math.floor(Math.random() * 6)) }
  ];

  // 예외 처리: 점수가 낮고 파일명에 노이즈가 있으면 미분류 폴더로 강제 이동
  var noisy = ['최종', '진짜최종', '수정본', 'v2', 'tmp']
    .some(x => fname.toLowerCase().includes(x));
    
  if (recs[0].score < 40 && noisy) {
    recs[0].folder = '미분류';
    recs[0].score  = 45;
  }

  // 결과 반환 예: [{folder: "재무/회계", score: 88}, {folder: "기술개발", score: 52}, ...]
  return recs;
}
```
